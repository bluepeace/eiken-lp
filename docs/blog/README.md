# ブログ（コンテンツマーケティング）記事

LP のテキストのもとになるコンテンツを管理するディレクトリです。

## WordPress での使い方（ブロックエディタでブロックに変換する）

記事本文（`posts/*.html`）は HTML で書いてあります。**そのまま貼ると「カスタムHTML」1ブロックになる**ため、見出し・段落・リストを**個別のブロック**にしたい場合は次の手順で変換してください。

1. 編集画面で **「ブロックを追加」→「クラシック」** を選ぶ
2. **クラシックブロックの中に**、HTML ファイルの本文（コメント以外）をすべてコピー＆貼り付け
3. クラシックブロックの右上 **「︙」（三点メニュー）→「ブロックへ変換」** をクリック

→ 見出し（h2/h3）・段落（p）・リスト（ul/ol）がそれぞれ「見出し」「段落」「リスト」ブロックに分割され、ブロックエディタでそのまま編集しやすくなります。

※ 変換後にレイアウトが少し崩れる場合は、該当ブロックだけ手で調整してください。

## 記事ファイルの書式（posts/*.html）

**タイトルをファイルの一番上に配置し、その下に記事本文を書く**形式に統一しています。

```
■ タイトル
（ここに投稿タイトルを1行で）

■ スラッグ
（WordPressの投稿スラッグ。英小文字・ハイフン。例：eiken-2026-schedule）

■ メタ説明例
（スニペット用の説明文）

※ （補足・確認事項があれば）

<!--
■ WordPressブロックエディタでの入れ方
...
========== 以下の本文を... ========== -->

<p>記事本文のHTML...</p>
```

今後も記事を作成する際は、**1行目から「■ タイトル」とタイトルを置き、その下にメタ説明・WordPress用メモ・本文**の順で記載してください。

## ディレクトリ構成

- **research/** … 記事作成前の調査メモ（ファクトベースの根拠を残す）
- **posts/** … WordPress にアップしやすい形式で整えた記事本文（先頭にタイトル）

## 記事の流れ

1. キーワードを決める
2. Web で情報収集し、`research/` に調査メモを残す
3. メモを元に `posts/` に記事を作成
4. 必要に応じて LP のテキストに反映

## トーン・ターゲット

- 受験者の保護者（特にお母さん）が読む想定
- 専門用語を避け、わかりやすく、親に寄り添う文章
- 各記事の最後に、英検対策アプリ AiKen（アイケン）の紹介を自然に入れる
